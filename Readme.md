角色：你是一位资深的软件架构师，擅长为本地离线应用设计高效、可维护的技术解决方案。你熟悉多种编程语言、数据库和UI框架，能够根据项目需求权衡技术选型的优缺点。

任务：
为一个本地离线的卡车健康管理软件设计技术栈。该软件需满足以下核心需求：

数据管理：
存储车辆信息（车牌号、里程数等）。
管理轮胎更换记录（10条轮胎，前4后6，需记录每条轮胎的更换日期和历史记录）。
管理机油更换、保养等维护记录。
支持添加备注（文本格式）。
功能要求：
数据持久化（本地存储，无需联网或登录）。
简单的UI界面，支持CRUD操作（增删改查）。
可导出数据（如CSV或Excel）。
非功能要求：
适用于Windows/macOS平台（桌面应用优先）。
开发效率高，维护成本低。
无需复杂的依赖管理。
问题：

推荐编程语言和UI框架，并说明理由（如Python + Tkinter、C# + WPF、Electron等）。
推荐数据存储方案（如SQLite、JSON文件、Excel等），并说明理由。
是否需要ORM（如SQLAlchemy、Entity Framework）？为什么？
有无推荐的开源库或工具可简化开发（如Pandas用于导出数据）？
如何确保数据完整性（如轮胎更换记录的校验）？
输出格式：

列出技术栈的每个组件（语言、UI、数据库等）及其理由。
提供简要的架构示意（如MVVM、MVC）。
列出潜在风险及缓解方案（如数据损坏、性能瓶颈）。
额外说明：

优先考虑开发速度和易用性，其次是性能。
无需支持移动端或Web。
